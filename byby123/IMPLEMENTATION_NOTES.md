# 구현 노트 및 강조 사항

이 문서는 구름 ICT 인턴십 채용 미션으로 수행된 실시간 채팅 애플리케이션의 주요 구현 정책, 기술 스택 활용 및 강조하고 싶은 부분들을 간략하게 설명합니다.

## 1. 핵심 기술 스택

*   **백엔드:** Node.js, Express, TypeScript, MongoDB (Mongoose), Socket.IO
*   **프론트엔드:** React, TypeScript, Bootstrap, React-Router-DOM, Socket.IO-Client, Axios

## 2. 아키텍처 및 통신 방식

*   **풀스택 분리:** 백엔드와 프론트엔드가 명확히 분리된 아키텍처로, 각각 독립적인 개발 및 배포가 가능합니다.
*   **RESTful API:** 사용자 인증, 대화 관리, 메시지 삭제 등은 RESTful API를 통해 처리됩니다.
*   **실시간 통신 (Socket.IO):**
    *   채팅 메시지 전송 및 수신, 파일 업로드 알림 등 실시간 상호작용에 Socket.IO를 활용했습니다.
    *   일반 채팅과 귓속말(1:1 대화)을 구분하여 메시지를 전송하고 수신할 수 있도록 Socket.IO Room 기능을 활용했습니다.

## 3. 주요 기능 구현 및 강조 사항

### 3.1. 실시간 채팅
*   **메시지 유지:** 브라우저 새로고침 또는 재접속 시에도 이전 채팅 내역이 유지되도록 구현했습니다.
*   **메시지 삭제:**
    *   사용자가 자신의 메시지를 개별적으로 삭제할 수 있습니다.
    *   모든 채팅 내역을 한 번에 삭제하는 기능도 제공합니다.

### 3.2. 파일 공유
*   **파일 업로드/다운로드:**
    *   `multer` 라이브러리를 사용하여 파일을 서버에 업로드하고, 업로드된 파일을 다운로드할 수 있는 기능을 구현했습니다.
    *   파일 업로드 시, 해당 파일 정보가 포함된 메시지가 실시간으로 채팅창에 표시됩니다.
*   **파일 시스템 연동:** 웹에서 파일 메시지를 삭제할 경우, 서버의 `uploads` 폴더에 저장된 실제 파일도 함께 삭제되도록 구현하여 불필요한 파일이 쌓이는 것을 방지했습니다.

### 3.3. 사용자 관리
*   회원가입, 로그인, 로그아웃 기능을 구현하여 사용자 인증을 처리합니다.
*   세션 기반 인증을 사용하여 사용자 상태를 관리합니다.

### 3.4. 상태 관리 (Context API 활용)
*   요구사항에 따라 별도의 상태 관리 라이브러리(Redux, Zustand 등) 없이 React의 내장 `Context API`를 활용하여 전역 상태를 관리했습니다.
*   `AuthContext`를 통해 로그인된 사용자 정보를, `SocketContext`를 통해 Socket.IO 인스턴스 및 연결 상태를 애플리케이션 전역에서 효율적으로 공유합니다.

### 3.5. UI/UX 및 스타일링
*   **Bootstrap 유틸리티 클래스:** `Bootstrap`의 강력한 유틸리티 클래스(flexbox, spacing, sizing, color, border 등)를 최대한 활용하여 일관되고 반응형인 UI를 구축했습니다. 이를 통해 커스텀 CSS/SCSS 코드의 양을 최소화하고 개발 효율성을 높였습니다.
*   **에셋 활용:** 제공된 아이콘 에셋(`history.svg`, `send.svg`, `upload.svg`)을 버튼에 적용하여 시각적인 명확성을 더했습니다.

## 4. 개발 정책

*   **TypeScript:** 프로젝트 전반에 TypeScript를 적용하여 코드의 안정성과 유지보수성을 높였습니다.
*   **모듈화:** 백엔드의 `routes`, `services`, `models` 및 프론트엔드의 `components`, `pages`, `context` 등 역할에 따라 코드를 모듈화하여 가독성과 재사용성을 확보했습니다.
*   **환경 변수:** `.env` 파일을 통해 API URL, 데이터베이스 연결 문자열 등 환경에 따른 설정을 분리하여 관리합니다.
